# 多用户联网 + AI 导航

Active Theory 最独特的两个特性：**实时看到其他访客** + **用自然语言对话浏览作品集**。

这两个功能在 5 个现有案例中均为零。它们代表了一个新的设计维度：**网站不再是一个人的体验**。

---

## 一、多用户联网光标

### 效果

当你访问 activetheory.net 时，页面顶部和底部会出现**彩色的管状粒子轨迹**——这些管的颜色和运动轨迹来自**其他正在访问网站的访客**。

### 技术架构

```
访客 A (浏览器)              服务器                 访客 B (浏览器)
     │                         │                         │
     ├─ mousemove ──────────► WebSocket ──────────► 渲染轨迹
     │                         │                         │
     │ 粒子系统                │ 广播                     │ 粒子系统
     │ (本地渲染)              │                          │ (本地渲染)
```

### 设计 Rationale

**为什么多用户光标有意义？**

1. **减少孤独感** — 个人网站浏览本质上是孤独的（一个人面对屏幕）。看到其他人的活动痕迹（"有人也在看这个"）创造了社区感。
2. **增加停留时间** — 你可能会多看几秒"看看其他人在做什么"。FWA 和 Awwwards 评委访问时会看到其他评委的光标——这变成了一个"隐藏社交网络"。
3. **差异化记忆点** — 用户访问过无数网站，但能记得"那个网站能看到其他人"的极少。这是一个强记忆锚点。

### 适合的场景

| 场景 | 推荐 | 原因 |
|------|------|------|
| **创意工作室网站** | ✅ | 展示技术能力 + 创造社区感 |
| **产品发布页** | ⚠️ 可选 | 制造"很多人同时在看"的热闹感 |
| **电商** | ❌ | 分散购买注意力 |
| **SaaS** | ❌ | 隐私顾虑 + 无必要 |
| **虚拟活动** | ✅✅ | 核心功能（和其他参与者互动） |

### 性能考虑

- 每个访客的状态需要序列化并通过 WebSocket 广播
- 大量并发（>1000）需要服务器端节流
- 粒子轨迹是本地 GPU 渲染的——服务器只传输坐标，不传画面

## 二、AI Chat 导航

### 效果

页面右下角有一个 AI Chat 输入框。你可以输入：

> "Show me a fun project"
> "Have you done any projects for crypto clients?"
> "What's the most technically impressive project?"

AI 会根据语义理解推荐对应的作品集项目，并直接导航过去。

### 技术架构

```
用户输入: "fun project"
     │
     ▼
  LLM (语义理解)
     │
     ├─ 识别意图: filter(projects, tags=["fun", "playful"])
     │
     ▼
  推荐结果: [项目 A, 项目 B]
     │
     ▼
  滚动到对应项目 + 高亮
```

### 设计 Rationale

**为什么 AI Chat 比传统筛选更好？**

1. **零学习成本** — 用户不需要理解"筛选分类体系"（什么是"Installations" vs "Experiences"?）。直接说人话即可。
2. **探索感** — 传统导航是"我知道我要找什么然后去找"。AI 导航是"我不知道我要什么，AI 帮我想"。这符合浏览作品集的心理——用户通常没有明确目标。
3. **品牌传递** — Active Theory 是技术工作室，AI 导航本身就是技术展示。"我们用 AI 了"这个信息不需要写在 About 页面——使用体验本身就在传递。

### 与分类筛选的对比

| 导航方式 | 用户意图 | 适合 |
|---------|---------|------|
| **传统菜单** | "我知道我在找什么" | 目标明确的用户 |
| **分类筛选** | "我知道我要找哪种类型" | 半目标用户 |
| **AI Chat** | "我不知道我要什么，帮我看看" | 浏览型用户 |
| **搜索** | "我知道具体名字" | 回头客 |

### 适合的场景

- **作品集 / 展示站** — 最合适（用户以探索为目的）
- **SaaS 功能页** — 可能（用户可以问"show me billing features"）
- **内容型网站** — 可能（"show me articles about WebGL"）
- **电商** — 谨慎（用户可能会问奇怪的问题，LLM 需要精心 prompt）

## 三、设计启示

### 从"单人浏览"到"共同在场"

ATv6 的两个特性（联网光标 + AI 导航）共同指向一个趋势：**网站正在从"一个人看一个页面"变成"一个人与一个系统/与他人互动"**。

| 阶段 | 模式 | 代表 |
|------|------|------|
| Web 1.0 | 静态阅读 | 文档站 |
| Web 2.0 | 单人交互 | 传统 SPA |
| **当前** | **单人 + 系统** | ATv6 (AI + 联网) |
| 未来 | 多人协同 | 虚拟空间 |

### 这些特性不适合所有网站

联网光标和 AI 导航在 ATv6 上有效，是因为：
1. **它是技术工作室的 portfolio** — 这两个特性本身就是作品的一部分
2. **访客以"欣赏技术"为目的** — 他们愿意花时间探索
3. **流量够大** — FWA/Awwwards 期间有足够的同时在线用户，联网光标才不会"荒凉"

如果你的网站是工具型或交易型，不要照搬。**每个效果都要匹配品牌叙事**。
