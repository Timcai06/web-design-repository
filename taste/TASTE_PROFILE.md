# TASTE PROFILE — 审美偏好档案

> Agent 执行任何 UI 任务前，必须先读此文件。
> 这里记录了项目负责人 Tim 的审美判断，是"好看"与"不好看"的最终裁决。

---

## 使用方式

每次你在 review 中给出审美判断（无论是评价一个 snippet、一个应用、还是一次改动），我将：
1. 把你的原话提炼为一条 **Taste Rule**
2. 归类到下方对应领域
3. Agent 下次生成时会遵守这些规则

每条 rule 包含：

```
Rule NNN: 规则名称
判断: "你的原话"
来源: snippet/app 名称 + 日期
改写: Agent 下次应该怎么做
```
