# Zentry — Design Rationale & Application Guide

Zentry 是 Resn 的"无 WebGL"技术宣言。它的设计哲学：**用 DOM 技术做出"看起来像 WebGL"的效果，但更快、更轻、更兼容。**

---

## 一、设计语言分析

### 视觉 DNA

```
高能量文字排版 ← 核心视觉
     ↓
Cinematic 视频叙事（双视频层）
     ↓
几何裁剪语言（clip-path portal / 菱形 / 多边形）
     ↓
HUD 风格 UI 元素（半透明、边框、科技感字体）
```

**为什么这套语言有效：**

1. **高能量文字（大写 + 粗体 + 逐字弹入）** — Zentry 是 Web3 游戏品牌，受众是年轻、高能量的玩家。文字动画的"爆发感"匹配品牌调性。如果换成优雅衬线体的 fade-in，Zentry 的品牌感就完全不一样了。

2. **双视频层（背景 + Portal）** — 这是"同屏双重叙事"：背景视频展示"世界的样子"，Portal 内视频展示"Portal 那边的世界"。用户在同一帧中看到"现实"和"可能性"——Web3 品牌的核心叙事（"现在的互联网" vs "Zentry 创造的互联网"）。

3. **Clip-path Portal 菱形** — 菱形不是矩形（屏幕惯性），也不是圆形（太柔和）。菱形有"科技感"和"动态感"——它的对角线暗示了方向和速度。选择菱形作为 portal 形状，是"打破屏幕 + 暗示动势"的几何最优解。

4. **HUD 导航** — 半透明、带边框、靠边浮动。源自战斗机 HUD（平视显示器）的设计语言——"信息叠加在视野上"而不是"信息占据一个区域"。这传达"未来感"和"科技感"。

### Resn vs Lusion 的设计哲学对比

| 维度 | Resn (Zentry) | Lusion (Oryzo / Luke) |
|------|--------------|----------------------|
| 技术选择 | DOM-first, WebGL-last | WebGL 为主 |
| 加载哲学 | "快就是好" — <2s 首屏 | "炫可以等" — 允许 3-5s 加载 |
| 动效风格 | 爆裂、冲击、戏剧（inOut） | 流畅、优雅、产品感（out） |
| 排版 | 大字、冲击、全宽 | 适中、留白、多列 |
| 色彩 | 暗色为主 + 高亮强调色 | 亮色为主 + 渐变色 |
| 适合 | 品牌站、SaaS、Web3 | 产品营销、创意 showreel |

**核心洞察：** Resn 风格适合"需要告诉用户品牌故事"的网站。Lusion 风格适合"需要展示产品本身"的网站。

## 二、每个效果的 Design Rationale + Application Guide

### 1. 3D Portal Mask (CSS Clip-Path)

**为什么好看：** 四边形的顶点从中心向四角展开，形成一个"从内向外爆裂"的视觉。因为展开路径是完全对称的，产生的效果是""

对比：
- 如果是"从上往下"展开 → 感觉像幕布拉开，仪式感强，但能量感弱
- 如果是"从左往右"展开 → 感觉像阅读翻页，自然而低调
- "中心向四角"展开 → 爆发、传送门、能量释放

**最适合的场景：**
- **品牌网站 Hero** — 进入时 Portal 展开 + 品牌名出现 = "进入我们的世界"
- **产品发布页** — Portal 展开揭示新产品
- **视频展示** — 用 Portal 做视频过渡

**不适场景：**
- 信息型网站 home（用户想快速找到内容，不需要戏剧性的 Portal）
- 长文本页面（文字不需要 Portal 效果）

**改编方案：**
```css
/* 五边形 Portal — 更有"科技感" */
clip-path: polygon(50% 0%, 100% 38%, 82% 100%, 18% 100%, 0% 38%);

/* 三角形 Portal — 更锐利、更有侵略性 */
clip-path: polygon(50% 50%, 100% 100%, 0% 100%);
```

### 2. Bento Grid + Perspective Tilt

**为什么好：** Bento Grid 是"非对称网格布局"——它让页面看起来像精心设计过的"桌面"，每个格子大小不一但彼此和谐。加上 perspective tilt hover 后，每张卡片都像"可以拿起来的实体物品"——增加了触觉感。

**适合放在：**
- **产品/服务展示**（SaaS 的三个核心功能、产品的三种配置）
- **团队介绍**（每人一张卡，点击展开详情）
- **作品集网格**

**改编方案 — Bento Grid 通用模式：**
```css
.bento-grid {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr;
  gap: 1rem;
}
.bento-grid__item--featured {
  /* 主打卡片占据更大空间 */
  grid-column: span 2;
  grid-row: span 2;
}
```

**Tilt 的性能注意：** 网格内卡片较多（>6）时，每个卡片都监听 mouse move 事件会导致大量计算。解决方案：使用事件委托在父容器上监听一次 mouse move，根据鼠标在父容器内的相对位置计算每个卡片的 tilt。

### 3. ScrollyVideo（滚动驱动视频）

**为什么好：** 传统视频播放是用户点击播放 → 视频以固定速度播放。ScrollyVideo 让**滚动速度决定视频速度**——这让用户感觉"我在控制视频，而不是视频在控制我"。

**适合放在：**
- **产品展示**（滚动速度与产品展示节奏同步）
- **品牌故事**（滚动前进带动故事视频推进）
- **教程/指南**（滚动到对应步骤，视频跳到对应画面）

**性能注意：** ScrollyVideo 需要视频编码为**关键帧每帧**（keyframe every frame），否则跳帧时会出现卡顿。建议使用 H.264 编码 + `-g 1`（每帧都是关键帧）。文件较大时（>20MB），优先用 WebM 格式或分多个短视频。

### 4. HUD 导航 + 滚动方向感知

**为什么好：** 大多数网站的导航栏是固定显示在顶部的。Zentry 的导航：
1. **向下滚动时隐藏** — 用户向下滚动是为了看内容，不是看导航。隐藏导航腾出更多屏幕空间给内容。
2. **向上滚动时显示** — 用户向上滚动时通常是在"回顾"或"寻找导航"——这时候出现导航正好满足意图。
3. **浮动在 Hero 上** — 导航不占据固定位置，而是浮在 Hero 内容上方——解决了 Hero 区域导航挡住内容的问题。

**适合放在：** 所有 scroll-driven 单页网站。这是"最佳实践级别的模式"，几乎可以无脑采用。

**改编方案：**
```javascript
let lastScroll = 0
window.addEventListener('scroll', () => {
  const currentScroll = window.scrollY
  if (currentScroll > lastScroll && currentScroll > 100) {
    nav.classList.add('nav--hidden')  // 向下 → 隐藏
  } else {
    nav.classList.remove('nav--hidden') // 向上 → 显示
  }
  lastScroll = currentScroll
}, { passive: true })
```

### 5. 音频可视化器

**为什么好：** 一个浮动导航里有一个"跳动的小条条"，完全与音频无关（导航里没有音频播放）。这是**纯粹的装饰性动效**——但它传达了一个信号："这个页面是活的"。

**设计原则：** 动效不需要"有用"——只需要"有道理"。一个跳动的小条条不需要真的有音频在播放，它作为一个视觉节奏器（visual metronome）已经完成了它的工作：让页面有"心跳"。

**适合放在：**
- 导航栏/侧边栏的小装饰
- Loading 动画
- 背景装饰元素
