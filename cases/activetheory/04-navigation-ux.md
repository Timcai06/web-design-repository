# Pillbox 导航 + 项目筛选

ATv6 的 UI 层极简——因为核心交互在 3D 世界中。UI 只做"恰好够用"的事。

---

## 一、Pillbox 导航

### 效果描述

一个极简的胶囊形导航，固定在页面角落。它响应滚动速度——快滚时缩小/淡出，慢滚时放大/显现。

```
[ Work · About · Contact ]   ← 平常状态
      [ · · ]                ← 快滚时（极小）
```

### Design Rationale

**为什么导航要响应滚动速度？**

1. **注意力分配** — 用户快速滚动时在"扫描内容"，不需要导航干扰。慢速滚动/停止时在"阅读内容"，这时候导航出现是合适的（用户可能想跳转）。
2. **沉浸感** — 3D 环境的沉浸感很容易被 UI 元素打破。动态显隐的导航在"需要时出现，不需要时消失"——最小化对 3D 世界的遮挡。
3. **极简即高级** — 一个只有 3 个链接的 pillbox 导航传递了"我们知道自己是谁"的信号。不需要复杂的 mega menu——"Work / About / Contact" 就是一个工作室网站的全部需求。

### 实现逻辑

```javascript
let lastScrollY = 0
let lastTime = Date.now()

window.addEventListener('scroll', () => {
  const now = Date.now()
  const dt = now - lastTime
  const dy = Math.abs(window.scrollY - lastScrollY)
  const velocity = dy / dt  // px/ms

  // 速度 > 1 px/ms → 缩小导航
  // 速度 < 0.3 px/ms → 恢复导航
  pillbox.style.opacity = velocity > 1 ? 0.3 : 1
  pillbox.style.transform =
    `scale(${Math.min(1, 1 - velocity * 0.3)})`

  lastScrollY = window.scrollY
  lastTime = now
}, { passive: true })
```

## 二、项目展示

### 效果描述

与常见的"网格缩略图 + 点击详情"不同，ATv6 的项目展示是**嵌入在 3D 环境中的**。当你滚动到项目区域时，3D 场景变为项目相关的视觉环境，项目信息以浮动面板形式出现。

### 项目筛选

项目支持两种筛选方式：

**1. 分类筛选（标签式）**
```
[ All · Games · Multiplayer · XR-VR-AI · Installations · Websites ]
```
点击标签 → 3D 环境过渡 → 只显示匹配项目

**2. AI Chat 筛选（自然语言）**
```
> "show me something fun"
→ 语义匹配项目 → 3D 场景过渡 → 高亮匹配项目
```

### Design Rationale

**为什么用 3D 环境展示作品而不是网格？**

1. **每个项目有自己的"世界"** — 网格展示把所有项目压平到同一尺寸和格式。3D 环境让每个项目拥有独特的视觉呈现——这更适合展示创意作品（每个作品都是独特的）。
2. **过渡即体验** — 从一个项目过渡到另一个时，3D 场景的变化本身就是"秀"。在传统网格中，切换项目只是"点击 → 新页面加载"。
3. **"做给你看"** — Active Theory 是 WebGL 工作室。如果他们的 portfolio 是用 WordPress 网格展示的，策略暗示就是"我们自己做的网站也不特别"。用 3D 环境展示作品本身就是能力证明。

### 与传统 Portfolio 的对比

| 维度 | 传统网格 (Dribbble/Behance) | ATv6 3D 环境 |
|------|---------------------------|-------------|
| 展示方式 | 截图 + 标题 | 3D 场景 + 浮动信息 |
| 项目间过渡 | 点击 → 刷新/路由 | 3D 场景过渡 |
| 信息密度 | 高（一屏看 6-12 个项目） | 低（每次看 1 个项目） |
| 适合 | 大量作品展示 | 精选作品（<20 个） |
| 视觉冲击 | 依赖单张截图 | 全屏沉浸体验 |

## 三、整体 UX 原则

### 极简 UI 层

ATv6 的 UI 层只有：
- Pillbox 导航（3 个链接）
- AI Chat 输入框
- 项目分类标签
- 滚动进度指示

没有任何多余的 UI 元素。**当你的 3D 足够强时，UI 可以退到几乎不可见。**

### 沉浸优先

```
沉浸度优先级:
  3D 环境 > 项目内容 > 导航 > 装饰性 UI

用户意图优先级:
  浏览作品 > 了解工作室 > 联系 > 其他
```

### 适合 / 不适合的场景

| 场景 | 适合 | 说明 |
|------|------|------|
| 创意工作室网站 | ✅✅ | 标准参考 |
| 个人作品集 | ⚠️ | 需要大量 3D 资源支撑 |
| 品牌站 | ⚠️ | 只有品牌调性匹配霓虹风格 |
| 产品站 | ❌ | 信息传递效率太低 |
| 电商 | ❌ | 3D 环境干扰购买路径 |
